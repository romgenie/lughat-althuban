# لغة الثعبان — Python العربية

لهجة برمجية من Python تكتب فيها الكلمات المفتاحية والدوال المدمجة والاستثناءات بالعربية. تُترجم ملفات `.apy` إلى Python القياسية في وقت التحميل وتُنفَّذ بواسطة CPython — دون تشعيب للمفسر.

**الحالة**: اكتملت Phase A بتاريخ (2026-04-19). تعمل واجهات الدخول الأربع: الملف، `-c` ، والمدخل القياسي (stdin)، والبيئة التفاعلية (REPL). تستورد وحدات `.apy` بعضها بعضاً. تطبع الاستثناءات غير المُعالجة تتبعات أخطاء بالعربية.
**Phase B**: لم يتم إقرارها بعد.
**الرخصة**: Apache-2.0.
**المستودع**: خاص خلال فترة ما قبل الإصدار.

## مثال سريع

```arabic
# hello.apy
دالة مرحبا(اسم):
    اطبع(f"مرحبا يا {اسم}")

لكل شخص في ["سارة", "أحمد", "ليلى"]:
    مرحبا(شخص)
```

```bash
$ ثعبان hello.apy
مرحبا يا سارة
مرحبا يا أحمد
مرحبا يا ليلى
```

## التثبيت

```bash
git clone https://github.com/GalaxyRuler/lughat-althuban
cd lughat-althuban
pip install -e .
```

يتطلب Python 3.11+. يُثبت سكربت ثعبان للأوامر.

## تشغيل الشيفرة

أربع واجهات دخول تماثل واجهة `python` القياسية:

```bash
ثعبان script.apy [args...]    # تشغيل ملف
ثعبان -c 'اطبع("مرحبا")'      # تنفيذ سلسلة نصية مباشرة
ثعبان - < script.apy          # القراءة من المدخل القياسي
ثعبان                          # البيئة التفاعلية REPL
```

`ثعبان --help` و `ثعبان --version` تعمل كما هو متوقع. رموز الخروج: `0` نجاح، `1` خطأ في التشغيل/الترجمة، `2` خطأ في الاستخدام.

## استيراد وحدات `.apy`

تُثبت الحزمة خطّاف استيراد في `sys.meta_path` بحيث يمكن استيراد ملفات `.apy` كأي وحدة Python:

```arabic
# main.apy
استورد helper
helper.مرحبا("عالم")
```

```arabic
# helper.apy
دالة مرحبا(اسم):
    اطبع(f"مرحبا يا {اسم}")
```

```bash
$ ثعبان main.apy
مرحبا يا عالم
```

تعمل الحزم المختلطة `.py` / `.apy` — ترى ملفات Python التي تستورد وحدات `.apy` أسماء المعرّفات المُترجمة والمُطبَّعة (على سبيل المثال: تستخدم `helper.مرحبا` من Python نفس الصيغة).

## تتبع الأخطاء بالعربية

تُطبع الاستثناءات غير المُعالجة بأسماء أنواع بالعربية ورسائل مُترجمة:

```bash
$ ثعبان -c '1 / 0'
تتبع_الأخطاء (المكدس الأحدث آخرا):
  ملف "<string>", سطر 1, في <الوحدة>
خطا_القسمه_على_صفر: القسمة على صفر
```

تمت ترجمة 38 نوعاً من الاستثناءات القياسية ونحو 30 رسالة مفسر شائعة. الأنواع والرسائل غير المُعرفة تمر دون تغيير.

## واجهة برمجة التطبيقات (API) العامة لـ Python

لتضمين اللهجة في أدوات أخرى:

```python
from arabicpython import (
    install,                       # تثبيت خطّاف استيراد .apy
    uninstall,
    run_repl,                      # بدء البيئة التفاعلية REPL
    install_excepthook,            # توجيه الاستثناءات غير المُعالجة عبر المترجم
    uninstall_excepthook,
    format_translated_exception,   # تنسيق ثلاثية (النوع، القيمة، تتبع الخطأ) بالعربية
)
```

## المعمارية في فقرة واحدة

تمر المصادر عبر `pretokenize` (طيّ الأرقام العربية إلى ASCII، ربط علامات الترقيم، رفض التحكم في اتجاه النص bidi) ← `tokenize` الخاصة بـ Python ← مُعيد كتابة الأسماء (NAME-rewriter) الذي يستشير قاموس عربي←Python القياسي ← `untokenize` ← `compile` ← `exec`. لا يوجد إعادة كتابة لشجرة AST ولا تشعيب لـ CPython. نفس المسار يدعم كل واجهات الدخول (CLI، REPL، خطّاف الاستيراد). راجع [`decisions/0001-architecture.md`](decisions/0001-architecture.md) للاطلاع على المبررات الكاملة و [`decisions/0004-normalization-policy.md`](decisions/0004-normalization-policy.md) لمعرفة كيفية طيّ متغيرات المعرّفات العربية (الهمزة، التاء المربوطة، الألف المقصورة).

## حدود معروفة (Phase A)

- **`from . import x` داخل `__init__.apy` للحزمة** لا تُترجم — يرى المُرَمِّز (tokenizer) `import` بعد `.` ويعاملها كعملية بحث عن خاصية. الحل البديل: `import pkg.sub as sub`. تم تأجيل الإصلاح إلى حزمة "إصلاحات الترجمة" المستقبلية.
- **الوصول إلى الخصائص عبر اللغات من .py إلى .apy** يجب أن يستخدم صيغة المعرّف المُطبَّعة وفق ADR-0004 (على سبيل المثال: `module.قيمه` وليس `module.قيمة`). هذا صحيح وفقاً لسياسة التطبيع ولكنه قد يربك المتعلمين الجدد.
- **لا يوجد اختبارات لسهولة استخدام async/await بعد.** الكلمات المفتاحية تُترجم، ولكن تغطية البرامج غير المتزامنة من البداية إلى النهاية ليست جزءاً من Phase A.

كل وحدة تنفيذ هي حزمة مواصفات. راجع [`specs/0000-template.md`](specs/0000-template.md) للتنسيق و [`specs/INDEX.md`](specs/INDEX.md) لسجل الحزم.

- **zhpy** (gasolin, 2014) — لهجة Python الصينية القائمة على الترميز المعجمي والتي يعتمد عليها هذا المشروع معمارياً.
- **قلب (Qalb)** لرمزي ناصر — النقد الذي يأخذه هذا المشروع بجدية، وهو السبب في جعل Phase B مرحلة متميزة ومؤجلة عمداً بدلاً من كونها الهدف الافتراضي.
- **Hedy** — دليل على أن التعليم المعتمد على Python متعدد اللغات وببنية تركيبية تدريجية يعمل على نطاق واسع.
