يستعرض هذا الدليل التعليمي لهجة apython بداية من "مرحبا بالعالم" وصولاً إلى استيراد وحداتك الخاصة. كل عينة شيفرة هي ملف `.apy` حقيقي يمكنك لصقه وتشغيله؛ معظمها يماثل العروض التوضيحية السبعة التصاعدية في [`examples/`](../examples/).

## 1. التثبيت

تتطلب apython إصدار Python 3.11 أو أحدث. من جذر المستودع:

```bash
git clone https://github.com/GalaxyRuler/apython
cd apython
pip install -e .
```

يُثبت ذلك سكربت `apython` للأوامر. تحقق من ذلك:

```bash
apython --version
```

لديك الآن أربع طرق لتشغيل شيفرة apython، تماثل واجهة `python` القياسية:

```bash
apython script.apy [args...]    # تشغيل ملف
apython -c 'اطبع("مرحبا")'      # تنفيذ سلسلة نصية مباشرة
apython - < script.apy          # القراءة من المدخل القياسي
apython                          # البيئة التفاعلية REPL
```

رموز الخروج: `0` نجاح، `1` خطأ في التشغيل/الترجمة، `2` خطأ في الاستخدام.

## 2. مرحبا بالعالم

احفظ هذا كـ `hello.apy`:

```arabic
اطبع("مرحبا، يا عالم")
```

شغّله:

```bash
$ apython hello.apy
مرحبا، يا عالم
```

هذا كل شيء. `اطبع` هو الاسم العربي لـ `print` في Python. كل الدوال المدمجة والكلمات المفتاحية وأنواع الاستثناءات الأخرى تعمل بنفس الطريقة — apython هي Python مع ترجمة جدول المعرّفات، لا أكثر.

توجد قائمة كاملة بكل الرموز المُترجمة في [`dictionaries/ar-v1.md`](../dictionaries/ar-v1.md). لست بحاجة لحفظها؛ فالأمثلة أدناه تقدم الكلمات المفتاحية حسب الحاجة.

## 3. المتغيرات والحساب

تعيين المتغيرات هو Python عادية — تقوم apython بترجمة الأسماء *المدمجة* فقط، وليس أسماءك الخاصة. اختر أي معرّف عربي يعجبك:

```arabic
العمر = 25
السنوات_حتى_التقاعد = 65 - العمر
اطبع(f"باقي {السنوات_حتى_التقاعد} سنة")
```

المخرج:

```
باقي 40 سنة
```

هناك بعض الأمور التي تجد الإشارة إليها هنا:

- **المعرّفات المكونة من عدة كلمات تستخدم `_`**، وليس المسافة. من المعتاد في العربية فصل المصطلحات المركبة بمسافة، ولكن مُرَمِّز (tokenizer) Python يعامل المسافة كحد فاصل، لذا تستخدم اللهجة الشرطة السفلية. هذه القاعدة عامة في apython.
- **تعمل f-strings بشكل طبيعي.** يتم ترميز التعبيرات الداخلية مسبقاً (pretokenized) تماماً مثل بقية المصدر، لذا فإن `f"{السنوات_حتى_التقاعد}"` تحلل المتغير تماماً كما تتوقع.
- **يتم طيّ الأرقام العربية-الهندية والشرقية تلقائياً.** كتابة `العمر = ٢٥` مطابقة لكتابة `العمر = 25`. خلط أنظمة الأرقام داخل ثابته عددية واحدة (مثلاً `١2`) يؤدي إلى `SyntaxError` — ترفض apython ذلك بوضوح بدلاً من التخمين.
- **يتم طيّ علامات الترقيم العربية تلقائياً** خارج السلاسل النصية: `،` ← `,`، `؛` ← `;`، `؟` ← `?`. لذا فإن `جمع(أ، ب)` مطابقة لـ `جمع(أ, ب)`. أما داخل السلاسل النصية، فيتم الحفاظ على كل محرف عربي كما هو.

راجع [`examples/02_arithmetic.apy`](../examples/02_arithmetic.apy).

## 4. مسار التحكم

الكلمات المفتاحية لمسار التحكم:

| Python | apython | تُقرأ كـ |
|---|---|---|
| `if` | `إذا` | "إذا" |
| `else` | `وإلا` | "وإلا" |
| `elif` | `وإلا_إذا` | "وإلا إذا" |
| `for` | `لكل` | "لكل" |
| `while` | `طالما` | "طالما" |
| `in` | `في` | "في" |
| `break` | `اكسر` | "اكسر" |
| `continue` | `استمر` | "استمر" |
| `range` | `نطاق` | "نطاق/مدى" |

حلقة مع شرط بداخلها:

```arabic
لكل عدد في نطاق(1, 6):
    إذا عدد % 2 == 0:
        اطبع(f"{عدد}: زوجي")
    وإلا:
        اطبع(f"{عدد}: فردي")
```

المخرج:

```
1: فردي
2: زوجي
3: فردي
4: زوجي
5: فردي
```

الإزاحة (Indentation) مهمة تماماً كما هي في Python — أربع مسافات لكل مستوى حسب الاصطلاح.

راجع [`examples/03_control_flow.apy`](../examples/03_control_flow.apy).

## 5. الدوال

كلمة `def` هي `دالة` (المصطلح الرياضي باللغة الفصحى لـ "function"). و `return` هي `ارجع`. تعمل الوسائط الافتراضية بالطريقة القياسية:

```arabic
دالة جمع(أ, ب=10):
    ارجع أ + ب

اطبع(جمع(5))         # 15
اطبع(جمع(5, 7))      # 12
```

أسماء الدوال، وأسماء المعاملات، والمتغيرات المحلية كلها من اختيارك — لا تترجمها apython، لذا يمكنك استخدام أي معرّف عربي أو إنجليزي أو مختلط طالما أنه اسم Python صالح.

ملاحظة حول تطبيع المعرّفات: تقوم apython بطيّ بعض متغيرات المحارف العربية إلى شكل قياسي واحد قبل البحث، لذا فإن `قيمه` و `قيمة` (بالتاء المربوطة) تشيران إلى نفس الاسم، وكذلك `إذا` و `اذا` (بالهمزة وبدونها). هذا يعني أن الإملاء الذي تكتبه مرن، ولكن الاسم المُخزن يكون بصيغة قياسية واحدة. القواعد الكاملة موجودة في [`decisions/0004-normalization-policy.md`](../decisions/0004-normalization-policy.md)؛ وبالنسبة للبرمجة اليومية يمكنك تجاهل هذه التفاصيل.

راجع [`examples/04_functions.apy`](../examples/04_functions.apy).

## 6. هياكل البيانات

القوائم والقواميس والصفوف والمجموعات كلها تعمل بشكل طبيعي. أسماء أنواعها مُترجمة (`قائمة` ، `قاموس` ، `صف` ، `مجموعة`) ولكنك نادراً ما تستدعي تلك المنشئات (constructors) مباشرة — فمعظم الشيفرات تستخدم الثوابت (literals):

```arabic
الفواكه = ["تفاح", "موز", "برتقال"]
الأسعار = {"تفاح": 3, "موز": 2, "برتقال": 4}

لكل فاكهة في الفواكه:
    اطبع(f"{فاكهة}: {الأسعار[فاكهة]} ريال")
```

المخرج:

```
تفاح: 3 ريال
موز: 2 ريال
برتقال: 4 ريال
```

التوابع الشائعة لها أسماء عربية: `.append ← اضف` ، `.pop ← انتزع` ، `.keys ← مفاتيح` ، `.values ← قيم_القاموس` ، `.items ← عناصر` ، `.get ← اجلب`. جدول التوابع الكامل موجود في القسم 6 من `ar-v1.md`.

راجع [`examples/05_data_structures.apy`](../examples/05_data_structures.apy).

## 7. الأصناف

كلمة `class` هي `صنف`. معامل `self` التقليدي هو `الذات`. التوابع السحرية (Dunder methods) مثل `__init__` لا تُترجم — فهي تحتفظ بإملائها في Python لأنها أسماء محددة في البروتوكول، وليست مفردات موجهة للمستخدم.

```arabic
صنف نقطة:
    دالة __init__(الذات, س, ص):
        الذات.س = س
        الذات.ص = ص

    دالة المسافة_من_الأصل(الذات):
        ارجع (الذات.س ** 2 + الذات.ص ** 2) ** 0.5

نقطة_1 = نقطة(3, 4)
اطبع(نقطة_1.المسافة_من_الأصل())   # 5.0
```

الوراثة، و `super()` (`الاصل()`)، و `classmethods` ، و `staticmethods` كلها تعمل بنفس الطريقة التي تعمل بها في Python — ترجم الكلمة المفتاحية، واكتب الباقي بأسلوب Python المعتاد.

راجع [`examples/06_classes.apy`](../examples/06_classes.apy).

## 8. الاستيراد

تُثبت apython خطّاف استيراد في `sys.meta_path` بحيث تستورد ملفات `.apy` بعضها البعض بشفافية:

```arabic
# helper.apy
دالة مربع(عدد):
    ارجع عدد ** 2

دالة مكعب(عدد):
    ارجع عدد ** 3
```

```arabic
# main.apy
استورد helper كـ مساعد

اطبع(مساعد.مربع(5))   # 25
اطبع(مساعد.مكعب(3))   # 27
```

شغّل `apython main.apy` وسيعثر الخطّاف على `helper.apy` ويترجمه ويجعله قابلاً للاستيراد.

الكلمات المفتاحية هنا هي `استورد` (`import`) و `كـ` (`as`). يمكنك أيضاً استيراد أسماء محددة: `من helper استورد مربع` (`from helper import مربع`).

وحدات المكتبة القياسية (`math` ، `json` ، `os` ، …) يتم استيرادها بأسمائها العادية في Python — `استورد math`. تقوم Phase A بترجمة اللغة فقط، وليس المكتبة القياسية. ربط أسماء المكتبات بأسماء مستعارة هو وظيفة Phase B؛ راجع [`decisions/0008-phase-b-charter.md`](../decisions/0008-phase-b-charter.md).

**الاستيراد عبر اللغات** يعمل في كلا الاتجاهين:

- يمكن لملف `.py` أن يستورد `helper` للحصول على وحدة `.apy`.
- يمكن لملف `.apy` أن يستخدم `استورد json` للحصول على وحدة مكتبة قياسية `.py`.

إحدى مشكلات Phase A: `from . import x` داخل `__init__.apy` للحزمة لا تُترجم بعد. الحل البديل: اكتب `استورد pkg.sub كـ sub`. هذا موثق في قسم "الحدود المعروفة" في ملف README الخاص بالمشروع ومقرر إصلاحه في حزمة مستقبلية.

راجع [`examples/07_imports.apy`](../examples/07_imports.apy).

## 9. قراءة رسائل الخطأ

تطبع الاستثناءات غير المُعالجة تتبعات أخطاء بالعربية:

```bash
$ apython -c '1 / 0'
تتبع_الأخطاء (المكدس الأحدث آخرا):
  ملف "<string>", سطر 1, في <الوحدة>
خطا_القسمه_على_صفر: القسمة على صفر
```

الهيكل هو نفسه تتبع أخطاء Python، فقط مع تسميات إطارات مُترجمة:

| Python | apython |
|---|---|
| `Traceback (most recent call last):` | `تتبع_الأخطاء (المكدس الأحدث آخرا):` |
| `File "..."` | `ملف "..."` |
| `line N` | `سطر N` |
| `in <module>` | `في <الوحدة>` |

تُترجم أسماء أنواع الاستثناءات بشكل منهجي. تصبح `ZeroDivisionError` هي `خطا_القسمه_على_صفر` ؛ وتصبح `KeyError` هي `خطا_المفتاح` ؛ وتصبح `ValueError` هي `خطا_القيمه`. السابقة `خطا_` تماثل اللاحقة `Error` في Python. هناك عدد قليل من الاستثناءات التي لا تنتهي أسماؤها في Python بكلمة `Error` تحتفظ بهذا التميز في العربية: `KeyboardInterrupt` ، `StopIteration` ، `Warning` ، `GeneratorExit` ، `SystemExit` لا تُسبق بـ `خطا_`.

القائمة الكاملة لجميع أنواع الاستثناءات الـ 38 المُترجمة موجودة في [`dictionaries/exceptions-ar-v1.md`](../dictionaries/exceptions-ar-v1.md).

عندما ترفع استثناءك الخاص، يمكنك استخدام اسم الصنف بالعربية أو بالإنجليزية — فكلاهما يؤولان إلى نفس الكائن:

```arabic
حاول:
    ارفع خطا_القيمه("القيمة غير صالحة")
استثناء خطا_القيمه كـ خطأ:
    اطبع(f"التُقط: {خطأ}")
```

المخرج:

```
التُقط: القيمة غير صالحة
```

تمت ترجمة حوالي 30 من رسائل المفسر الأكثر شيوعاً أيضاً (`division by zero` ، `list index out of range` ، `unsupported operand type(s)`). الرسائل التي ليس لها ترجمة تمر دون تغيير بالإنجليزية — فرسالة إنجليزية صادقة أفضل من رسالة عربية مضللة.

## 10. إلى أين تذهب بعد ذلك

- **شغّل الأمثلة.** الأمثلة من [`examples/01_hello.apy`](../examples/01_hello.apy) إلى `07_imports.apy` هي عروض توضيحية غنية تصاعدياً. كل منها قصير عن قصد؛ ومعاً يمارسون كل ميزة في اللهجة.
- **تصفح القاموس.** [`dictionaries/ar-v1.md`](../dictionaries/ar-v1.md) هو قائمة الكلمات القياسية. لست بحاجة لحفظه، لكن تصفحه لمدة 5 دقائق يبني خريطة ذهنية مفيدة لما هو مغطى.
- **اقرأ ADR المعمارية.** إذا كنت فضولياً بشأن كيفية حدوث الترجمة، فإن [`decisions/0001-architecture.md`](../decisions/0001-architecture.md) يشرح تصميم المعالج المسبق من المصدر إلى المصدر في صفحة واحدة.
- **جرب REPL.** فقط شغّل `apython` بدون وسائط. إنه يقبل نفس بنية الكلمات المفتاحية العربية مثل السكربت، مع إدخال متعدد الأسطر عبر مطالبات استمرار.
- **سجل المشكلات (issues) للأمور التي فاجأتك.** لقد اكتملت ميزات Phase A ولكن القاموس سيستمر في التطور في `ar-v2`. إذا كانت إحدى الترجمات تبدو غريبة، فهذه إشارة تستحق التسجيل.

إن apython هي لهجة تعليمية أولاً وأداة إنتاج ثانياً (راجع [`decisions/0007-scope.md`](../decisions/0007-scope.md) للترتيب الصريح). شكل Phase B — بما في ذلك ما إذا كان `استورد طلبات` سيعني في النهاية `import requests` — موضح في [`decisions/0008-phase-b-charter.md`](../decisions/0008-phase-b-charter.md).
