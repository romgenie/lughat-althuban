# Spec Packet 0006: cli-v1

**Phase**: A
**Depends on**: Packet 0005 (translate); ADR 0001 (architecture), ADR 0002 (file extension), ADR 0007 (scope).
**Estimated size**: medium (one focused implementer session)
**Owner**: Gemini 3.1 Pro

## Goal

Implement the `apython` command-line runner: read a `.apy` file (or `-c` code, or stdin), translate it via Packet 0005, compile, and execute. After this packet lands, a learner can install the package and immediately run `apython hello.apy` from a terminal.

The script entry point `apython = "arabicpython.cli:main"` is already declared in `pyproject.toml`. This packet implements `arabicpython/cli.py` so that entry point works.

## Non-goals

- **No REPL.** Interactive mode is a separate packet (Phase A scope per ADR 0007 but not this packet).
- **No import hook.** Importing `.apy` modules from inside other `.apy` files is a separate packet.
- **No `-m module` flag.** `-m` requires the import hook to resolve `.apy` modules; it's blocked on that packet. Stock `python -m` for `.py` modules is also out of scope here — users who want that should use stock `python`.
- **No translated tracebacks.** Exception messages remain English. Translated traceback layer is a separate packet.
- **No bytecode caching.** `.apyc` is reserved (per `.gitignore`) but not produced in v1.
- **No source maps for tracebacks.** Tracebacks point at the translated Python line/col, not the original `.apy` line/col. The `compile(..., original_path, "exec")` call gives correct *file* attribution; line numbers may drift ±1 per ADR 0001.
- **No watch/reload mode.** Runs once and exits.
- **No multi-file batch execution.** One file per invocation.

## Public interface

### Entry point

```python
def main(argv: "list[str] | None" = None) -> int:
    """apython CLI entry point.

    Args:
        argv: command-line arguments NOT including program name. If None,
            defaults to sys.argv[1:].

    Returns:
        Exit code: 0 on success, 1 on translate/compile/runtime errors,
        2 on usage errors (mirrors argparse convention).
    """
```

### Command-line surface

```
apython [-h] [--version] [-c CODE] [FILE] [args ...]
```

Modes (mutually exclusive; argparse enforces):

- **File mode**: `apython hello.apy [args ...]`
  - Read `hello.apy` as UTF-8 text.
  - Translate, compile with `original_path = abspath(hello.apy)`.
  - Execute in a fresh module namespace with `__name__ = "__main__"`, `__file__ = abspath(hello.apy)`, `__builtins__` = stdlib `__builtins__`.
  - Set `sys.argv = ["hello.apy", *args]` for the duration of the run, restore afterward.

- **Inline mode**: `apython -c "اطبع('hello')" [args ...]`
  - Treat the `-c` argument string as source.
  - `original_path = "<string>"`, `__file__` = `"<string>"`.
  - `sys.argv = ["-c", *args]`.

- **Stdin mode**: `apython - [args ...]`  *(`-` as the FILE positional)*
  - Read source from `sys.stdin.read()`.
  - `original_path = "<stdin>"`, `__file__` = `"<stdin>"`.
  - `sys.argv = ["-", *args]`.

- **`--version`**: print `apython <version>` (the `__version__` from `arabicpython/__init__.py`) and exit 0.
- **`-h` / `--help`**: print usage and exit 0.

If no FILE and no `-c`: print usage to stderr, exit 2 (argparse default).

### Argument forwarding

Anything after the FILE (or after `-c CODE`) is passed through as `sys.argv[1:]` for the executed program. The runner itself consumes only its own flags.

For argparse, this means `parse_known_args` won't suffice — use a custom parser or set `nargs=argparse.REMAINDER` on the trailing positional. Implementation hint: argparse's `REMAINDER` is the cleanest for "everything after the file is the program's argv."

### Error reporting

All errors print to `stderr` and produce a non-zero exit code. Format mirrors stock `python` where reasonable:

| Condition | Message format | Exit code |
|---|---|---|
| File not found | `apython: can't open file '{path}': [Errno 2] No such file or directory` | 1 |
| File not readable (perm denied) | `apython: can't open file '{path}': {exception}` | 1 |
| `SyntaxError` from translate (bidi, mixed digits, unclosed string) | The `SyntaxError`'s own message, prefixed with `File "{path}", line L:` if line info available | 1 |
| `SyntaxError` from `compile` of translated output | Standard Python traceback formatting via `traceback.print_exc()` | 1 |
| Exception during `exec` | Standard Python traceback via `traceback.print_exc()` | 1 (or `SystemExit.code` if the program called `sys.exit(N)`) |
| `KeyboardInterrupt` during `exec` | Print `KeyboardInterrupt` to stderr | 130 (matches stock python) |

The runner does NOT translate exception messages in v1.

### `__main__` execution semantics

Execute via:

```python
import sys, types
mod = types.ModuleType("__main__")
mod.__file__ = original_path
mod.__loader__ = None
mod.__package__ = None
mod.__spec__ = None
sys.modules["__main__"] = mod  # save and restore the previous one
old_argv = sys.argv
sys.argv = [original_path, *forwarded_args]
try:
    exec(code, mod.__dict__)
finally:
    sys.argv = old_argv
    # Restore previous sys.modules["__main__"] if there was one.
```

This mirrors what `runpy.run_path` does, minus the import hook. Do not use `runpy` directly — it would re-tokenize as `.py` and miss our pipeline.

## Implementation constraints

- **Dependencies**: stdlib only (`argparse`, `sys`, `os`, `types`, `traceback`, `pathlib`).
- **Python version**: 3.11+.
- **No global state mutation outside the executed program**: save/restore `sys.argv` and `sys.modules["__main__"]` around the `exec` call.
- **Encoding**: read source files as UTF-8 explicitly (`open(path, encoding="utf-8")`). Do not rely on platform default. If the file is not valid UTF-8, raise a clean error with the path.
- **Style**: pass `ruff check .` and `black --check .` at project defaults (line length 100).
- **Testability**: `main` takes `argv` so tests can call it directly without subprocess. `main` returns an int — tests check the return value rather than catching `SystemExit`. The script entry point is a thin shim that calls `sys.exit(main())`.

## Test requirements

All tests in `tests/test_cli.py`. Use exact test names below. Pytest only.

Tests should mostly call `main(argv=[...])` directly, capturing stdout/stderr with `capsys` and using `tmp_path` for fixture files. **One** subprocess test (`test_subprocess_end_to_end`) verifies the actual entry point works.

### Help and version (3)

1. `test_help_flag_exits_zero`: `main(["-h"])` raises `SystemExit(0)` (argparse behavior). Catch the exception, assert `.code == 0`.
2. `test_version_flag_prints_version`: `main(["--version"])` exits 0 and prints a string containing `apython` and the version from `arabicpython.__version__` to stdout.
3. `test_no_args_prints_usage_exits_two`: `main([])` exits with code 2 and writes usage info to stderr.

### File mode — golden path (5)

4. `test_run_simple_arabic_file`: write `examples/hello.apy` content (`اطبع("مرحبا")\n` or whatever the dictionary maps to `print`) to `tmp_path / "hello.apy"`, call `main([str(path)])`, assert exit 0 and stdout contains `مرحبا`.
5. `test_run_pure_ascii_file`: a file containing `print("hi")\n` runs and prints `hi`. Verifies the runner doesn't break on ASCII-only input.
6. `test_run_arabic_keywords_and_digits`: a file with Arabic for `if x > ٥: print(x)` and a setup line `x = ١٠` runs and prints `10`.
7. `test_run_function_def_and_call`: a file defining a function in Arabic and calling it; assert expected output.
8. `test_run_class_with_method`: a file defining a class in Arabic and using it; assert expected output.

### Inline mode (2)

9. `test_inline_arabic_code`: `main(["-c", "اطبع('xy')"])` exits 0 and prints `xy`.
10. `test_inline_with_forwarded_args`: `main(["-c", "import sys; print(sys.argv)", "a", "b"])` prints a list whose first element is `-c` and which contains `a` and `b`.

### Stdin mode (2)

11. `test_stdin_arabic_code`: `monkeypatch` `sys.stdin` to a `StringIO` containing Arabic source, then `main(["-"])`; assert it executes.
12. `test_stdin_with_forwarded_args`: same but verify `sys.argv` inside the run is `["-", "extra"]`.

### `__name__`, `__file__`, `sys.argv` (4)

13. `test_dunder_name_is_main`: file containing `print(__name__)` runs via file mode, prints `__main__`.
14. `test_dunder_file_is_absolute_path`: file containing `print(__file__)` runs and prints the absolute path of the .apy file.
15. `test_sys_argv_in_file_mode`: file containing `import sys; print(sys.argv)` invoked as `main([str(path), "alpha", "beta"])` — first element is the path (absolute or as-passed; spec the choice and stick to it), and `alpha`/`beta` are present.
16. `test_sys_argv_restored_after_run`: store `sys.argv` before, call `main([str(path)])`, assert `sys.argv` is unchanged after.

### Error reporting (6)

17. `test_missing_file_prints_error_exits_one`: `main(["does_not_exist.apy"])` returns 1 and stderr contains `can't open file` and the path.
18. `test_directory_instead_of_file`: `main([str(tmp_path)])` (a directory) returns 1 with a clean error.
19. `test_bidi_in_source_reports_syntax_error`: file with `\u202E` outside any string — `main` returns 1 and stderr contains `bidi control` and `trojansource.codes`.
20. `test_mixed_digit_in_source_reports_error`: file with `١2` — returns 1 and stderr mentions `mixed digit`.
21. `test_translated_python_syntax_error`: a file whose translated form is invalid Python (e.g., dangling colon) — returns 1, stderr contains `SyntaxError`.
22. `test_runtime_exception_prints_traceback`: a file with `1/0` — returns 1, stderr contains `ZeroDivisionError` and `Traceback`.

### `sys.exit` and `KeyboardInterrupt` (3)

23. `test_program_calls_sys_exit_zero`: file containing `import sys; sys.exit(0)` — `main` returns 0.
24. `test_program_calls_sys_exit_nonzero`: file containing `import sys; sys.exit(7)` — `main` returns 7.
25. `test_keyboard_interrupt_exits_130`: simulate by raising `KeyboardInterrupt` from inside the executed code (e.g., `raise KeyboardInterrupt`); `main` returns 130 and stderr mentions `KeyboardInterrupt`.

### File encoding (2)

26. `test_utf8_with_bom_handled`: a file written with a UTF-8 BOM still runs correctly. (Decide: strip the BOM before passing to `translate`, or rely on `tokenize` to handle it. Document the choice.)
27. `test_non_utf8_file_clean_error`: a file containing bytes that are not valid UTF-8 — `main` returns 1 and stderr mentions encoding/decoding without a Python traceback.

### Examples directory (1)

28. `test_examples_hello_runs`: invoke `main([str(repo_root / "examples" / "hello.apy")])` (locate via `pathlib.Path(__file__).parents[1]`), assert exit 0 and some expected stdout. This exercises the real shipped example, ensuring it doesn't bit-rot.

### Subprocess smoke (1)

29. `test_subprocess_end_to_end`: use `subprocess.run([sys.executable, "-m", "arabicpython.cli", "-c", "اطبع('ok')"], capture_output=True, text=True, encoding="utf-8")`. Assert returncode 0 and stdout contains `ok`. Verifies the module is importable and the entry point works in a fresh interpreter.
