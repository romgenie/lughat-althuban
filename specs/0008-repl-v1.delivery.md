## What shipped — files created, key implementation choices

- `arabicpython/repl.py`: Implemented the `ArabicConsole` class and `run_repl` function.
    - `ArabicConsole` extends `code.InteractiveConsole` and overrides `runsource` to translate Arabic source to Python using the existing `translate` pipeline before compilation and execution.
    - Added logic to distinguish between "incomplete input" (e.g., unclosed blocks or strings) and "real translate-time errors" (e.g., bidi rejection or mixed-digit literals) by inspecting `SyntaxError` messages, mirroring CPython's `code.compile_command` behavior.
    - `run_repl` sets up a best-effort `readline` environment (history and tab completion), initializes the prompt environment variables (`APYTHON_PS1`, `APYTHON_PS2`), and starts the interactive loop.
- `arabicpython/cli.py`: Integrated the REPL by modifying the "no arguments" execution path to drop into an interactive session.
- `arabicpython/__init__.py`: Re-exported `run_repl` for public API access.
- `tests/test_repl.py`: Added 27 comprehensive tests for the REPL, including multi-line construct handling, error reporting without tracebacks for syntax errors, and environment variable overrides.
- `tests/test_cli.py`: Updated to verify that running `apython` with no arguments drops into the REPL.

## Deviations from the spec — anything you did differently and why; "None" if verbatim

- **`test_multiline_class_def` logic**: The spec suggested that two blank lines might be needed to exit a nested `class/def` block. In practice, stock Python's `InteractiveConsole` (and `code.compile_command`) sees a single blank line as enough to terminate the top-level compound statement in `symbol="single"` mode. I updated the test expectation to `False` for the first blank line to match this standard behavior.

## Implementation notes worth remembering — non-obvious decisions

- **Incomplete Input Detection**: Since `translate()` converts `tokenize.TokenError` into `SyntaxError`, I implemented `_is_incomplete_marker` to look for specific strings like `"EOF in multi-line statement"` which are used by `tokenize` to signal that more input is required.
- **`__init__.py` Cleanup**: I consolidated the re-exports in `arabicpython/__init__.py` to use explicit `as` aliases, which satisfies `ruff`'s `F401` check for unused imports while keeping them available for library users.
- **Readline Fallback**: The REPL setup is fully defensive against missing `readline` or `rlcompleter` modules, ensuring a graceful degraded experience on Windows or other systems without a GNU-compatible readline.

## Planner addendum (2026-04-18, post-merge)

Merged as squash commit [`3d5502d`](https://github.com/GalaxyRuler/apython/commit/3d5502d). Initial CI was 7-of-9 red on two test failures, both rooted in spec defects. Planner pushed [`a458fad`](https://github.com/GalaxyRuler/apython/commit/a458fad) directly to the branch to resolve them rather than round-trip:

1. **Unclosed-string spec defect.** Spec test 4 (`test_runsource_returns_true_for_unclosed_string`) used `s = 'unclosed` and expected `runsource` to return `True` (incomplete). But single-quoted unclosed strings are **not** continuation candidates in Python — stock `python` REPL gives a `SyntaxError`, not a continuation prompt. Only triple-quoted strings, parenthesized expressions, and explicit-block constructs are incomplete. Fix: changed the test to use `s = """unclosed`, which is genuinely incomplete and matches Gemini's `_is_incomplete_marker` substrings (`"EOF in multi-line string"` on 3.11, `"unterminated triple-quoted string literal"` on 3.12+).

2. **Atexit-history spec defect.** Spec asserted "atexit ensures we save even on EOF" — wrong. `atexit` handlers fire at process exit, not when `console.interact()` returns normally inside a still-running pytest process. Test 26 (`test_history_file_used_when_readline_present`) failed on Linux/macOS for this reason (Windows skipped because no readline). Fix: `_setup_readline` now returns the resolved history path, and `run_repl` saves explicitly in a `finally` block. The `atexit` registration is kept as a fallback for crash scenarios.

Both fixes are mechanical and squarely planner-attributable (the spec dictated wrong expectations). Same precedent as Packet 0005's missing-skipif fix: when the issue is "spec gap, not implementer miss," push directly. Re-CI was 9-of-9 green within ~1 min.
